GET /media?type=image|video&q=&from=&to=&page= — non-deleted media for the active profile, newest first, paginated; each item carries key, cdnUrl, dimensions, bytes, and referenceCount. Thumbnails/URLs are derived from the DB key + one CloudFront host — no assets/media/... mount, no per-caller rewriting (fix-on-rebuild: single media host).

POST /media (presigned) — returns { key, uploadUrl }; the browser PUTs bytes straight to S3, then finalizes the row (images/videos). No NFS mount, no file_exists()/->move(), no "directory not writable" on a missing prefix (fix-on-rebuild: S3 media storage).

DELETE /media/:id — server runs the single reference-count query first; returns 409 Conflict + the usage list when the file is still referenced (post / campaign / funnel). The physical S3 delete is never keyed off one feature's table (fix-on-rebuild: media ownership / reference count).
